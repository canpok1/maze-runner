## 手順

1. 会話セッションでの作業内容を振り返る（実施タスク、使用したスキル/エージェント、問題点、スムーズだった点）

2. `.claude/`配下の設定ファイルを確認する（`CLAUDE.md`、`agents/`、`skills/`）

3. 振り返り内容と現在の設定を照らし合わせて分析する
    - **改善点**: 曖昧な指示、手順の問題、役割分担の不明瞭さ、不足スキル、設定との乖離
    - **よかった点**: 機能したパターンが明文化されているか、暗黙知を設定に反映すべきか

4. 改善が必要と判断した場合、内容に応じて対応する
    - 些細な修正（誤字脱字）: ユーザー確認の上で直接修正
    - それ以外: GitHub Issue作成（作成前に重複チェック）
        1. `github`スキルで作成予定タイトルを検索し、open状態のIssueを確認
        2. 類似Issueがある場合: 作成をスキップし、既存IssueのタイトルとURLを表示
        3. 類似Issueがない場合: `github`スキルでIssueを作成
            - **タイトル**: 簡潔で具体的な内容
            - **本文**: 改善点の場合は問題点/改善提案/期待効果、よかった点の場合はパターン説明/設定への反映内容/明文化の効果
            - **注意**: Issue作成時に `--label` オプションを使用しないこと。ラベル付与は `optimizing-issue-labels` スキルの責務である

5. 改善不要の場合は、Issue作成せずユーザーに報告（よかった点は共有）

## 注意点

- 改善点は具体的かつ実行可能な形で提案
- 複数の改善点は独立したIssueとして作成
- 主観的な好みではなく、実際の問題や非効率性に基づいて判断
- 設定の意図を理解し、既存方針を尊重
