## 概要

GitHub Issueの内容が不十分な場合に、Issueの詳細化・明確化を行うワークフローです。

## 手順

1. GitHub Issueの番号をユーザーに確認する。
    - 引数で指定されている場合、この手順はスキップする。
2. ghコマンドでGitHub Issueの現在の内容を把握する。
3. Issueの内容を分析し、不足している情報を特定する。
    - 背景・目的が明確か
    - 現状の問題点が具体的に記述されているか
    - 期待する結果が定義されているか
    - 受け入れ条件が明確か
    - 技術的な制約や考慮事項が記載されているか
4. `document-specialist` エージェントに依頼して、精緻化したIssue内容を作成する。
5. 精緻化した内容をユーザーに確認し、承認を得る。
6. 承認後、ghコマンドでIssueを更新する。

## 注意点

- 元のIssueの意図を変えないこと。
- 追加情報が必要な場合は、ユーザーに確認すること。
- 技術的な判断が必要な場合は、推測せずにユーザーに質問すること。
