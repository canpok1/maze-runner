description: タスクIssueの実装からPR作成・レビュー対応・振り返りまでを一括実行する
argument-hint: [GitHub IssueのURLまたは番号]
---
## 手順

1. GitHub Issueの番号を決定する。
    - 引数で指定されている場合、その番号を使用する。
    - 引数が未指定の場合は、Issue番号が必要であることをユーザーに伝えて作業終了。
2. `/develop` コマンドの手順を実行する。
    - 指定されたIssue番号を引数として渡す。
    - 実装・テスト・自己レビューを完了する。
3. `/create-pr` コマンドの手順を実行する。
    - プルリクエストを作成する。
4. `/fix-pr` コマンドの手順を実行する。
    - CI状態の確認とレビューコメント対応を完了条件を満たすまで実施する。
5. `/retro` コマンドの手順を実行する。
    - 作業セッションを振り返り、必要に応じてGitHub Issueを作成する。

## 注意点

- 各ステップ間でユーザー確認を求めず、自律的に進行すること。
- `/fix-pr` の完了条件を満たすまで、全ての対応を継続すること。
- エラーや想定外の状況が発生した場合は、その時点でユーザーに報告して判断を仰ぐこと。
